# 今时今日你第一次写文章，是否仍然秉持关于创建世界级技术文档的同样7 条准则？规则有没有演变？

ok，下面是七条规则：

1. 枯燥是原罪

2. 开始动笔之前，思考清楚你想让读者在阅读后收获什么

3. 框架完整的大纲，每次都有。

4. 避免模棱两可的代词

5. 清晰 = 配图+文字

6. 当涉及概念解析....有逻辑的配图和示例

7. 欣然接受修改

对我影响最大的规则是 #1：枯燥是原罪。
